Core metadata: description

The description of a registered resource is free text (in English, please). The description should be readable prose, and should contain keywords by which your registry entry might be found in a full-text search. (Q.v. the separate subject field where more keywords can be added.

The description should be in plain text, not HTML. Line breaks may be included for basic formatting but not all viewing software will reproduce these.

Recommended practice is to enter here a single, concise paragraph and keep the full description of the resource on the WWW: see the reference URL field in the core metadata.
